Trojitá

Why Trojitá?

  • Efficient workflow for Devs on Desktops
  • Very fast at fetching mail from IMAP mailboxes
  • Low resource mode for costly, slow networks
  • Built to be used on Mobile Phones
    • easily portable to mobile UIs
    • Ubuntu Dekko
  • Upcoming support for Secure Messaging
  • Use with your own/org's IMAP server

What is Trojitá?

a fast, Qt IMAP Client

Fast

  • HackerNews commenter
    • ~15 Gb of email on IMAP server
    • "blazing fast"

Qt

  • Cross Platform
    • Tried on Windows, Linux, Nokia Harmattan
  • Great library support for further feature extension
  • Runs with Qt5 as well
  • Extensible UI using QML
  • KDE project, but no dependecies on KDE Libraries
  • Lightweight
    • Arch Linux: 5.6 Mb
    • Windows: 4.4 Mb + libraries

IMAP

  • Internet Message Access Protocol
    • Succeeded POP
  • Multiple Clients connected simultaneously
  • Server Side Searches
  • Downloads MIME parts
  • more..

MIME: Multi-Purpose Internet Mail Extensions

Trojitá is very compliant with the IMAP Spec. It in fact was built by the author during his BSc and MSc Thesis work around IMAP extensions

Architecture

  • Model-View arch

  • Qt framework (C++)

  • Very modular, layered architecture with clear interfaces

  • Pluggable GUI layers (QtGUI, QML)

  • Lazy, on-demand loading

  • Streams layer multiple interfaces

  • Unit-test suite

A Two Line Patch

BUG: Message editor should prompt to save on ANY change, not only body

I just spent several minutes individually pasting about 30 e-mail addresses into a message, which I intended to save as a draft for later reuse. Because I made several changes (adding several To fields and editing the subject line), I expected that clicking Cancel would ask if I wanted to save the draft.

Instead, Trojita sees the empty message body, assumes there is nothing to save, and then throws away the last few minutes of my work.
